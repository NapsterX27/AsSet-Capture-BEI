How to use the portal

A quick, click-through guide to the Site Inventory Portal. Pick your role below — the steps expand as you go. Nothing here changes any data; it's safe to explore, and every Open button jumps straight to the real screen.

You don't need a key or a login. Open the link, find what you need, and — if something's wrong — ask for a change in a couple of taps. Your view is always read-only, so you can't break anything.

Delivery status at a glance

1Find a site & its assets

Open Assets and choose your job site. Then search by Unit # or Serial #, and narrow the list with the trade / status filters.

  • The filters you set go into the page's link — copy the URL to share the exact filtered view with a teammate.
  • Use Export CSV to pull the current list into Excel.
  • A Pending tag on an asset means a change request is already open for it.
Open Assets
2Ask for a change (reassign / report an issue)

Found an asset assigned to the wrong trade, or something that needs attention? On the asset, choose Reassign trade or Report issue, add a note, and submit. No GitHub account needed.

Your request goes to the admins to action — you'll see the Pending tag appear on that asset within about a minute.

Open Assets
3Track a delivery

Open Deliveries. Active shows orders still in progress; Completed is history. Click any tracker to expand its lines.

Each line shows: Part #, description, Required date, Expected qty, Delivered, Pending (Expected − Delivered), Picked up, who picked it up, and a status (see the legend above).

A tracker with an Overdue line gets an Overdue badge in its header, so late orders stand out in the list.
Open Deliveries
4Find things fast & print a pickup sheet

On Deliveries:

  • Search by Req #, trade, or originator.
  • Filters (shows a count when active) opens a compact bar: filter by Trade, Originator (shown as "By NAME"), and Date — a one-click Today or an exact From / To required-date range.
  • The list pages 100 at a time — use the page picker above it.
  • Expand a tracker and click Print sheet for a clean, branded pickup sheet to carry into the field.
Open Deliveries

Admins do everything the team can, plus create and update trackers, publish inventory, and manage who else has access. All of it is gated by your admin key.

1Sign in

Open Admin → Sign in, enter your name and your admin key. There are two kinds of key, and both work the same way:

  • Master key — the account owner's key (set once in Cloudflare). Always works.
  • Team key — a personal key an admin generated for you under Team access (step 2).
Your sign-in is remembered on this device across the Assets, Deliveries and Admin tabs — you won't be asked again each visit. Sign out to require the key again.
Open Admin
2Manage team access (add / revoke admins)

On Admin, the Team access card lists everyone who can sign in.

  • + Add admin → enter a name → the portal generates a one-time key and shows it once. Copy it and give it to that person; they sign in with it (step 1).
  • Remove next to a name revokes their access immediately.
You never touch Cloudflare for this — the list lives in the repo, and keys are stored hashed (never in plain text). The Master key row is always there and can't be removed here.
Open Admin
3Publish inventory (Equipment Master)

Under Admin → Equipment Master (or Assets → Import Equipment Master), upload the latest .xlsx export. It's parsed in your browser — the file is never uploaded, only the built inventory JSON is committed.

Check the preview counts, then Publish. The site refreshes for everyone in a minute or two.

Open importer
4Create delivery trackers from a Req

Open Deliveries → New tracker and drop the Requisition .xlsx. It's read in your browser by exact column header (order can vary) and grouped into one tracker per Order Number.

  • Assign a Trade to each order (or Set trade for all). If unsure, pick Other and fix it later (step 6).
  • Exclude any old/completed order you don't want tracked.
  • Orders already tracked are auto-hidden and skipped, so you never add one twice.
  • Very large orders (200+ lines) are split into "Part 1 of N" trackers automatically.
  • Re-drop a file later and use Backfill originator to add the Transaction Originator onto trackers created before.

Then Create N trackers — creation is paced and resumable, so large imports (hundreds of orders) finish reliably.

Open tracker form
5Log deliveries & pickups

Expand a tracker. Each line has three inline controls in the log cell:

  • Expected — the target qty. It prefills from the file when the export includes Quantity To Receive; otherwise type it and click Set. The line then shows Pending = Expected − Delivered.
  • Deliver — when material arrives, enter the qty delivered and click Log.
  • Pickup — when the team takes material, enter the qty and the pickup person's name, then Log.

Status moves Not started → On site → Partial pickup → Picked up. When every line is fully picked up, the tracker auto-closes to Completed. Your name and the date are recorded on each entry.

Open Deliveries
6Corrections & housekeeping
  • Fix a mistyped pickup qty: under Picked up by, click Edit next to the entry, correct the number, and Save (it asks you to confirm).
  • Logged too much/little: log an adjusting delivery/pickup (a negative qty subtracts).
  • Change a trade after creation: use the Trade dropdown in the tracker header.
  • Mark complete: if the export omits already-handled lines, click Mark complete on the header to move it to Completed.
  • Delete a tracker: expand it and click Delete order (it leaves the app; the GitHub Issue is closed as a record).
  • Purge legacy: old-format trackers show a Purge N legacy button on the Active toolbar.
Open Deliveries
7Good to know
The repo is public. Trackers hold material descriptions, quantities, Req #, project and ship-to — never pricing or costs. Don't put anything confidential in a Req line.
  • List freshness: a new or closed tracker can take a few seconds to appear for other viewers (GitHub's index) — you see your own changes instantly.
  • Setting up a separate project? The whole portal can be cloned for another environment with its own data and admins — see CLONING.md in the repo.
